gerenciando repositórios github. aula 05 capítulo 18 módulo 03

aplicativo do github. ao clicar em abrir com visual estudio code, após selecionar o repositório, o visual estudio code será aberto e qualquer alteração no código será indicado no botão lateral, no aplicativo github também mostrará que o arquivo foi alterado e o que eraantes de alteração e como ficou após a alteração. no aplicativo do github em comit, para salvar essa alteração, digitar um título e uma descrição. clicar o commit to master. neste caso foi atualizado apenas no repositório local. para atualizar o repositório remoto: no aplicativo github botão push origin, onde origin é tudo que for remoto, repositório remoto. clicar e esperar, e o arquivo foi atualizado no remoto.

para criar um novo repositório e o botão não aparecer, clicar em "file" depois em "new repository" ou comando "ctrl+n" e repetir os passos da aula 04 do capítulo 18 módulo 03, orém alterando os nomes e a pasta a ser criado o repositório. o aplicativo do github cria uma pasta no computador, caso o repositório escolhido na criação não exista e tem um campo onde escolhe com qual repositório se vai trabalhar. no site do github esolher o repositório e clicar em settings para alterar alguma configuração, no final da lista de opções tem como deixar público ou private.

para deletar um repositório remoto: no aplicativo github selecionar o repositório a ser apagado, clicar em "repository" em seguida clicar em "remove". marcar para enviar para lixeira e clicar em apagar. para deletar um repositório remoto: no site do github, escolher o repositório a ser deletado ir em configurações "settings" desce até o final da página, na "denger zone", e clicar em "delete this repository" confirmar e pronto.


hospedando site gratuitamente com github pages. aula 06 capítulo 18 módulo 03

pode adicionar arquivos no repositório local com control+c control+v. o arquivo principal deve ser de nome index. uma um projeto for único ao publicar com o github pages, ao invés de escolher um tema, no campo "source" escolher "master branch", e já será criado o link e o site já está hospedado.

para gerar um qr code do site; entrar no site: qr-code-generator.com selecionar "url" e colar o endereço do site hospedado. clicar em "generate". pode escolher bordas, mas algumas são pagas. ao clicar em download, mesmo que apareça uma tela pedido "free sign up" esperar até o download iniciar sozinho.

esta hospedagem do github é apenas para html css e javascript não aceita nodejs, php ou outros. mas a publicação de códigos, repositório, é qualquer linguagem.

imagens que se repetem no fundo do site. aula 03 capítulo 19 módulo 03

para configurar o tamanho da imagem de fundo: backgroud-size: valoraltura valorlargura; a imagem irá encolher ou crescer, porém ao crescer pode perder resolução e começar a ficar embaçada

a repetição é automática, para que a imagem não repetida no backgroud: backgroud-repeat: no-repeat; , para repetetir apenas uma linha ou coluna, usar: backgroud-repeat: repeat-x ou repeat-y;. alterar o tamanho da viewport não altera o tamanho da imagem.


configurando a posição da imagem de fundo do site. aula 04 capítulo 19 módulo 03

pode-se deifinir qual o posicionamento se inicia o alinhamento de um backgroud imagem usado: background-position: posição posição; na posição de adiciona o horizontal esquerda direita left rigth e vertical acima abaixo top bottom, sendo esta a marcação de qual ponto se inicia a exibição da imagem de fundo. posicionamento padrão acima top esquerda left. ao deifinir primeiro coluna depois linha

altura da viewport vh, área visível do navegador, 100% da viewport: height: 100vh; 100% da viewheigth


mudando o tamanho da imagem de fundo do site. aula 05 capítulo 19 módulo 03

o padrão dao background-size é auto. opção contain: altera o tamanho da imagem de background de acordo com a largura da área visível, porém na altura não preenche toda viewport, se alterar a altura mesma coisa porém nas lateris. opção cover: corta as laterais ou acima e abaixo da imagem de background para se adaptar ao tamanho da viewport

background attachment e shorthand. aula 06 capítulo 19 módulo 03

o padrão do background attachment é scroll: o conteúdo rola junto com o fundo da tela. opção fixed: o conteúdo rola, porém a imagem de background continua fixa em uma posição, a ancoragem de fixação da posição depende do atributo em background-position e vai ser considerado de acordo com a área visível da viewport.

um shorthand agrupa em uma linha várias configurações, no caso do background deve ser na ordem: color > image > position > repeat > {size} > attachment que fica: background: cor url('caminho') center center no-repeat cover fixed; . na documentação do background diz que esse shorthand precisa ter o tamanho, porém ao usar não funciona, possivelmente os navegadores não se adaptaram ainda, neste caso retirar a configuração de tamanho {size} do shorthand e adicionar o backgroud-size em outro linha: background: cor url('caminho') center center no-repeat fixed; backgroud-size: cover; resolvendo assim o problema

centralização vertical de caixas. aula 07 capítulo 19 módulo 03

por padrão todo box em html tem posição relativa: position: relative; . quando o position recebe a configuração absolute é possível configurar a posição do objeto, top rigth left bottom. usando top: 100px em uma caixa de posicionamento absoluto essa caixa vai ter uma distância de 100 pixel da borda da caixa pai onde ela se encontra, ou o conteiner, pode usar top, bottom, left, rigth seguido de valor, ou se usar uma porcentagem, top: 50%; left: 50%;, o canto superior esquerdo será posicionado em 50% da caixa pai tanto pelo topo quanto pela esquerda ficando assim no centro, mas o conteúdo da caixa não ficará centralizado e sim o canto. usando o atributo transform pode-se alterar configurações específicas de uma caixa, como rotação, escala, translate entre outros, o translate mudará a posição da caixa mesmo centralizado com o canto, definindo valor horizontal e vertical entre parênteses: transform: translate(20px 20px);

margin auto só serve para alinhamento centralizado na horizontal linha. para alinhar na vertical coluna: adicionar a caixa que se quer alinhar ao centro o posicionamento absoluto left e top e transform: position: absolute; transform: translate(-50% -50%); ou configurar o bottom o rigth e o translate com valor positivo. posicionamento do pai relativo, posicionamento do filho absoluto
